## 1. 每日开始检查

**频率：** 每工作日 08:00 执行

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 访问 `/admin/alerts` | 告警数 = 0 或仅有 info 级告警 |
| 2 | 检查 `/admin/system-health` | 所有服务指标为绿色 |
| 3 | 查看当日待处理申诉 `/admin/appeals` | 无逾期未处理申诉（> 3 工作日） |
| 4 | 检查待审超学分申请 `/admin/pending-overloads` | 无积压超过 48 小时的申请 |
| 5 | 确认候补升级邮件已发送 | 查看最近 24 小时审计日志 |

**告警分级处理优先级：**
- 🔴 error：同日内必须处理
- 🟡 warning：24 小时内处理
- 🔵 info：本周内跟进

## 2. 学期生命周期管理

### 2.1 创建新学期

1. 进入 `/admin/terms`，点击 **新建学期**
2. 填写：
   - 学期名称（如 `2026秋季学期`）
   - 开始日期 / 结束日期
   - **注册开放时间** / **注册截止时间**
   - 退课截止日期（Drop Deadline）
3. 保存后，学期状态为 `INACTIVE`

### 2.2 开启注册窗口

1. 在学期列表点击 **编辑**
2. 将 `registrationOpen` 切换为 `true`
3. 确认注册优先级窗口（大四优先，依次开放）：
   - 大四：注册开放日 D+0
   - 大三：D+2
   - 大二：D+4
   - 大一：D+6
4. 在 `/admin/reg-windows` 验证优先级分组显示正确

### 2.3 关闭注册窗口

1. 注册截止日期到达时，系统自动停止接受新注册
2. 手动确认：编辑学期 → `registrationOpen = false`
3. 通过 `/admin/alerts` 确认 `registration-closed` 状态

## 3. 选课注册窗口操作

### 3.1 开放流程

```
新学期公告发布 (T-14天)
    ↓
教学班创建完毕 (T-7天)
    ↓
课程目录开放浏览 (T-3天)
    ↓
注册窗口开放 (T=0，按年级分批)
    ↓
退课截止 (T+2周)
    ↓
注册窗口关闭
```

### 3.2 处理超出学分限制申请

1. 进入 `/admin/pending-overloads`
2. 逐条审阅学生当前学分负载（系统自动计算）
3. 批准条件参考：
   - 在读 GPA ≥ 3.5
   - 无有效学籍限制 (Hold)
   - 所申请超学分 ≤ max+3
4. 点击 **批准** 或 **拒绝**，系统自动发送邮件通知

### 3.3 紧急关闭某教学班

1. `/admin/sections` → 找到对应教学班
2. 将 `capacity` 改为当前已选人数（座位=0 即停止自动加入候补）
3. 在公告系统发布关班通知

## 4. 学生账户管理

### 4.1 新生入学账户创建

**批量导入（推荐）：**

1. 准备 CSV 文件，表头：`email,studentId,legalName[,password]`
2. 进入 `/admin/students` → **批量导入**
3. 上传 CSV，预览错误行
4. 确认无误后提交，系统自动：
   - 创建用户账户
   - 发送邮箱验证邮件
   - 生成初始密码（默认 `Student123!`）

**单个创建：**

1. 生成邀请码：`/admin/invite-codes` → 新建邀请码
2. 将邀请码发给学生，学生自行注册

### 4.2 账户锁定解除

若学生因密码错误 8 次被锁定：

1. `/admin/students` → 搜索学生
2. 点击 **安全** 标签页
3. 点击 **解锁账户**
4. 可选：同时点击 **重置密码** 发送重置邮件

### 4.3 添加学籍限制 (Hold)

1. `/admin/holds` → **新建限制**
2. 选择学生、类型（FINANCIAL / ACADEMIC / REGISTRATION / DISCIPLINARY）
3. 填写原因和过期时间（如不填则永久有效直到手动解除）
4. 保存后学生将无法进行选课操作

## 5. 成绩录入与关闭

### 5.1 成绩录入流程

1. 教学节结束后，进入 `/admin/grade-entry`
2. 选择学期 → 选择教学班
3. 批量或逐条填写成绩（A/A-/B+/B/B-/C+/C/C-/D/F/W/I）
4. 点击 **保存**，系统发送成绩公告邮件给学生

**成绩标准对照：**

| 字母成绩 | 绩点 | 说明 |
|---------|------|------|
| A+ / A  | 4.0  | 优秀 |
| A-      | 3.7  | 接近优秀 |
| B+      | 3.3  | 良好+ |
| B       | 3.0  | 良好 |
| B-      | 2.7  | 良好- |
| C+      | 2.3  | 一般+ |
| C       | 2.0  | 一般 |
| C-      | 1.7  | 一般- |
| D       | 1.0  | 及格 |
| F       | 0.0  | 不及格 |
| W       | —    | 退课（不计入GPA） |
| I       | —    | 未完成（不计入GPA） |

### 5.2 成绩申诉处理

1. `/admin/appeals` 查看待处理申诉
2. 联系对应教师核实
3. 在申诉详情页填写决定说明
4. 点击 **批准** 或 **拒绝**
5. 若批准：在 `/admin/grade-entry` 修改成绩
6. 系统自动邮件通知学生

**SLA：** 成绩申诉应在 **5 个工作日** 内完成处理

## 6. 候补名单管理

### 6.1 自动升级机制

当有学生退课时，系统**自动**将候补队列第一位学生升级为已选课状态，并发送邮件通知。

管理员无需手动操作，但需在 `/admin/alerts` 确认升级邮件正常发送。

### 6.2 手动清理候补队列

若教学班已结束且仍有候补学生：

1. `/admin/waitlist` → 筛选对应教学班
2. 批量选中候补学生
3. 点击 **批量移出候补**（状态改为 DROPPED）
4. 发送公告通知受影响学生

### 6.3 候补分析

通过 `/admin/waitlist-analytics` 查看：
- 各教学班候补深度
- 候补转化率
- 部门/课程候补热点

## 7. 申诉与豁免处理

### 7.1 先修课豁免 (Prereq Override)

工作流：学生提交 → 导师审批 → 教师审批 → 管理员终审

1. `/admin/prereq-waivers` 查看待终审申请
2. 确认前两步已完成（审批历史中有导师和教师签字）
3. 审阅豁免理由（学生需说明具体理由）
4. **批准**：学生可加入对应教学班（无需满足先修要求）
5. **拒绝**：填写拒绝原因（学生可见）

### 7.2 超学分申请 (Credit Overload)

单步审批，仅需管理员：

1. `/admin/pending-overloads` 查看
2. 审阅学生当前总学分、GPA、申请理由
3. 系统显示批准后学分总数预览
4. 批准/拒绝，自动邮件通知

## 8. 邀请码管理

### 8.1 批量生成邀请码

1. `/admin/invite-codes` → **批量生成**
2. 设置：
   - 数量（建议每次不超过 200 个）
   - 最大使用次数（默认 1 次）
   - 有效期（建议 30 天）
3. 导出 CSV 后分发给招生办

### 8.2 CSV 批量导入

如果需要指定邀请码（如与外部系统同步）：

1. 准备 CSV，表头：`code[,maxUses,expiresAt]`
2. `/admin/invite-codes` → **CSV 导入**
3. 系统自动跳过已存在的 code

### 8.3 停用已泄露邀请码

1. 在邀请码列表找到目标 code
2. 点击 **停用**（`active = false`）
3. 已使用的注册不受影响

## 9. 系统告警响应

### 9.1 告警分类及处理

| 告警 ID | 描述 | 处理方式 |
|---------|------|---------|
| `grade-appeals` | 待处理成绩申诉 | 进入申诉页面处理 |
| `missing-grades` | 已完成教学班缺成绩 | 联系教师或在 grade-entry 补录 |
| `active-holds` | 有效学籍限制数量 | 正常，如过多则检查是否有误操作 |
| `near-capacity` | 教学班接近满员（≥ 90%）| 考虑增开教学班 |
| `not-closed-out` | 上个学期未完成关账 | 执行学期关账操作 |
| `waitlist-overflow` | 候补队列异常深 | 检查是否需要增加座位 |

### 9.2 自动刷新

告警页面每 60 秒自动刷新。若告警突然消失，可能是：
1. 问题已自动解决（如学生退课触发候补升级）
2. 数据缓存失效（刷新页面确认）

```bash

```

## 11. 学期结束关账

### 11.1 关账前检查清单

- [ ] 所有教学班成绩已录入
- [ ] 无待处理成绩申诉
- [ ] 先修课豁免申请已全部决定
- [ ] 超学分申请已全部决定
- [ ] 候补队列已清空或处理

### 11.2 执行关账

1. 进入 `/admin/closeout`
2. 选择目标学期
3. 点击 **预览关账**，系统显示影响人数
4. 确认无误后点击 **执行关账**
5. 系统将：
   - 所有 ENROLLED 状态改为 COMPLETED
   - 关闭注册窗口
   - 锁定成绩修改
6. 执行完成后，在审计日志确认 `term_closeout` 记录

### 11.3 关账后操作

1. 生成学期总结报告（`/admin/cohort-analytics`）
2. 下载毕业资格报告（`/admin/graduation`）
3. 归档该学期数据（可选：导出完整 CSV 包）
4. 为下一学期准备教学班

## 12. 应急处理

### 12.1 系统无法访问

```bash
# 检查 API 服务状态
curl https://api.yourdomain.com/health

# 检查容器状态
docker ps

# 重启服务
docker-compose restart api web
```

### 12.2 数据库连接失败

1. 检查 PostgreSQL 服务状态
2. 确认 `DATABASE_URL` 环境变量正确
3. 检查网络/防火墙规则
4. 参见 `docs/disaster-recovery.md` 完整恢复流程

### 12.3 邮件发送失败

症状：学生未收到注册确认邮件、候补升级通知等

1. 检查 `/admin/system-health` 邮件服务状态
2. 确认 `SMTP_HOST/PORT/USER/PASS` 环境变量
3. 在设置页发送测试邮件
4. 临时方案：在审计日志查找受影响用户，手动联系

### 12.4 学生大量反映无法登录

可能原因：
1. **账号未验证**：学生未点击验证邮件 → 在 `/admin/students` 搜索并告知重发验证邮件
2. **账号被锁定**：解锁操作见 § 4.2
3. **服务器时间偏差**：检查 JWT 有效期配置
4. **会话超时配置**：默认 2 小时 Access Token，30 天 Refresh Token

### 12.5 成绩被错误批量修改

1. 立即在 `/admin/audit-logs` 查找操作记录
2. 确认操作时间和 actorUserId
3. 根据审计记录逐条恢复原成绩
4. 如涉及大量数据，联系数据库管理员执行回滚
