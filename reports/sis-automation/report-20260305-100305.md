# SIS 自动化推进报告（2026-03-05 10:03:05 MST）

## 1. 本轮变更
- 变更范围严格控制为：1 个后端硬化 + 1 个前端修复。

### 后端硬化（1/1）
- 文件: `apps/api/src/main.ts`
- 改进:
  - 新增 `resolveTokenName` 与 token 名称白名单校验（仅允许 RFC token 字符）。
  - 对 `CSRF_COOKIE_NAME`、`CSRF_HEADER_NAME` 做启动期合法性校验，非法值自动回退到 `csrf_token` / `x-csrf-token`。
  - 记录结构化 `config_warning` 日志，便于运维定位错误配置。
- 价值:
  - 避免 CSRF 名称被错误或异常字符污染导致安全校验失效或误拒绝。

### 前端修复（1/1）
- 文件: `apps/web/lib/api.ts`
- 改进:
  - 新增与后端一致的 token 名称解析与合法性校验逻辑。
  - CSRF 名称读取顺序统一为：`NEXT_PUBLIC_*` -> `CSRF_*` -> 默认值。
  - CSRF header 名称统一标准化为小写，避免跨环境大小写漂移。
- 价值:
  - 浏览器端与后端 CSRF 命名规则一致，降低 403（`CSRF_TOKEN_INVALID`）概率。
