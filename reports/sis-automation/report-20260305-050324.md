## 遗留风险
- API 无法连通数据库，关键链路（登录、注册、找回密码、选课）运行态不可验。
- 虽然本轮已关闭响应体中的敏感链接回传，但邮件系统本身的投递成功率仍未在运行态闭环（受 DB/运行环境阻塞）。

## CTO 视角：问题与改进
- 问题：当前主要风险已从“代码漏洞”转向“环境不可运行导致无法验收”，使生产化推进节奏受阻。
- 改进：
  - 提供可用 Postgres 环境（或放开 Docker 权限）作为自动化基线前置条件。
  - 将 `AUTH_EXPOSE_DEBUG_LINKS` 在部署模板中固定为 `false` 并加入发布前检查，防止配置漂移导致敏感链接泄露回归。

## 用户视角：问题与改进
- 问题：用户端不再暴露重置/激活链接是正确方向，但当前仍无法真正完成登录与业务流程（后端不可用）。
- 改进：
  - 先恢复 API+DB 可用性，再执行一次完整登录/找回密码/注册链路验收。
  - 邮件文案可继续加入“未收到邮件时联系支持”入口，提升找回路径可达性。

## 阻塞原因与所需输入
